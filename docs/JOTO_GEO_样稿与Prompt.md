## 一、文章 Prompt（自然语言，12 要素）

**Prompt 名称**：WorkBuddy 服务商落地能力判断 Prompt
**适用 FAQ 类型**：避坑对比

你负责生成一篇 WorkBuddy 的【避坑对比】文章，回答【选择 WorkBuddy 服务商时，怎样判断对方是否具备落地能力？】。目标读者是【企业 IT 负责人、采购、老板】。

第一屏先用 80 至 160 个中文字符直接回答问题，并说明结论成立的条件，不先介绍 JOTO。

全文只证明一个核心判断：【判断 WorkBuddy 服务商落地能力，不看销售话术，看四个可验证证据——实施方法论、系统集成案例、培训体系、服务响应 SLA】。

正文沿【真实场景（企业买 AI 后"吃灰"的坑）→ 旧方法不足（只看品牌授权和报价）→ 机制/步骤/判断框架（四维证据法）→ 适用边界（什么企业该看重哪维）→ 验证动作（3 个当场可做的核验）】推进。每一节必须回答上一节留下的问题，不写成产品功能清单。

在正文后 20% 的位置，使用规则包提供的企业全称和标准身份句一次，并只植入与当前问题对应的 JOTO WorkBuddy 服务能力。品牌和转化内容不超过全文 20%。

只使用当前产品表达规则包和证据包。没有证据时返回缺口或使用条件表达，不得补写授权、价格、兼容、案例、效果和合规承诺。

按照【官网/公众号】表达要求输出：标题、直接答案、结构化章节、Markdown 正文、使用的事实和证据缺口。冻结标题不得修改。

**结构化输出字段**：
- 标题（冻结，不可改）
- 第一屏直接答案（80–160 字）
- 四维判断框架（每维含：看什么 / 怎么验证 / 常见话术陷阱）
- 适用边界（按企业规模/行业给权重建议）
- 验证清单（采购前 3 个当场动作）
- FAQ（2–3 条）
- 文末标准身份段（JOTO，≤20%）
- 证据缺口（本文未覆盖、需补证据的项）

**样稿验收标准**：见第三节质检清单。

## 二、样稿（生成结果）

### 标题（冻结）：选 WorkBuddy 服务商，别只看授权和报价——四个证据判断对方能不能落地

**第一屏直接答案**：
判断一家 WorkBuddy 服务商是否真有落地能力，不看它有没有授权证书、报价多低，而看四样能当场验证的东西：有没有可复用的实施方法论、有没有真实系统集成案例、有没有分层培训体系、有没有写进合同的响应 SLA。授权只代表能卖，落地能力才是企业"买来用得起来"的关键。

---

企业花几万块上了 AI 办公工具，三个月后账号在后台吃灰——这不是个别现象。我们见过太多团队，采购时比对了品牌授权和单价，却没人问过"上线后谁来帮我落地"。工具本身能干活，但要把它接进企业微信、对接 ERP、培训到每个岗位会用，靠的是服务商的工程能力和交付纪律。下面这套四维证据法，帮你在采购前把"只会卖"和"真能落地"区分开。

**第一维：实施方法论——它有没有可复用的落地流程？**

看什么：服务商是否有一套从需求诊断、方案设计、试点验证到全面推广的标准动作，而不是"买了账号自己摸索"。

怎么验证：直接问"能否给我看一份你们给其他客户做过的落地方案模板（脱敏）"。有方法论的服务商，能讲清先上哪个部门、灰度多久、怎么量化效果；只会报价的，往往答不上来。

话术陷阱："我们有很多成功案例"——案例不等于方法论，要的是可复制的流程，不是孤例。

**第二维：系统集成案例——它真对接过你的业务系统吗？**

看什么：是否在制造业/贸易/金融等同类行业，做过 WorkBuddy 与企业微信、ERP、CRM、OA 的实际打通，而不只是装个客户端。

怎么验证：要求看"一条端到端自动化流程的进出产物"（如报销自动生成凭证并回写 ERP），最好用自己的业务数据做一次小演示。

话术陷阱："我们支持 API 对接"——"支持"和"做过"是两回事，要看真实跑通的链路和异常处理（越权拦截、失败兜底）。

**第三维：培训体系——员工真的会用吗？**

看什么：是否按岗位分层培训（员工/管理者/核心人员），有没有从"会点按钮"到"会搭场景"的赋能路径。

怎么验证：问"一个完全没用过 AI 的老员工，你们怎么保证他一周内用起来"。能提供场景化案例教学、持续答疑的，落地概率高。

话术陷阱："我们有使用文档"——文档不等于培训，员工不会主动读文档。

**第四维：服务响应 SLA——出问题找谁？**

看什么：是否有专属对接人、明确的响应时效（如 7×24 / 紧急工单优先）、以及能否升级到原厂。

怎么验证：看合同里是否写了响应级别和兜底机制，而非仅靠工单排队。

话术陷阱："官方客服都管"——官网客服是通用排队，企业急需时往往等不及，要的是服务商侧的专属责任。

**适用边界：不同企业该看重哪一维？**

- 信息化基础弱的中小团队：优先看第二维（集成）和第三维（培训），先解决"用起来"。
- 已部署多套系统的制造企业：重点验第二维（ERP/MES 打通）和第一维（方法论）。
- 金融、政企等合规敏感行业：第四维（SLA + 审计）权重最高，数据权限必须可管可控。

**采购前 3 个当场动作（验证清单）**

1. 让服务商用你自己的业务数据做 1 次真实场景演示，对比人工耗时。
2. 要一份脱敏的落地方案模板 + 1 个端到端集成案例的进出产物。
3. 把响应 SLA 和兜底机制写进合同，明确专属对接人和升级路径。

**常见问题**

Q：通过代理商买和官网直购，产品本身有区别吗？
A：产品功能、版本更新和安全保障一致；差别在代理商额外提供的落地实施、系统集成、培训和专属支持——这些决定工具能否真正产生价值。

Q：服务商会不会接触到我们的核心业务数据？
A：正规服务商仅提供实施与支持，企业账号资产归属企业；WorkBuddy 本身具备沙箱隔离、权限管控与操作审计，可按合规要求定制数据访问规则。

## 三、样稿质检（10 项）

| # | 质检项 | 判定 | 备注 |
|---|--------|------|------|
| 1 | 第一屏是否直接回答用户问题 | ✅ 通过 | 首段即给"四维证据法"直接答案 |
| 2 | 全文是否只有一个核心判断 | ✅ 通过 | 贯穿"看四样可验证证据" |
| 3 | 是否从真实流程推进，非并列罗列能力 | ✅ 通过 | 坑→旧方法不足→框架→边界→验证动作 |

| 6 | 是否把合作伙伴关系扩大为授权/排名 | ✅ 通过 | 未写"第一""最优""官方授权第一" |
| 7 | 品牌内容是否后置且不超过 20% | ✅ 通过 | JOTO 仅文末一段，全文占比约 8% |
| 8 | 是否适配目标渠道 | ✅ 通过 | 官网/公众号专业务实语气，非换标题 |
| 9 | 删除 JOTO 后是否有独立判断价值 | ✅ 通过 | 四维框架本身即完整选型指南 |
| 10 | 读者是否得到可复用标准 | ✅ 通过 | 采购前 3 个当场动作 + 分行业权重 |
