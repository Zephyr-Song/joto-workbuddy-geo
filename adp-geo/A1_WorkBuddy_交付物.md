## 六、单题内容规范清单（A1 核心交付物②）

> 每行四要素：**核心作答重点 / 必填内容 / 内容边界 / 禁忌要点**。

### Q1 · WorkBuddy 是什么 / 与个人版区别
- **核心作答重点**：企业级 AI 智能体工作台；能自主规划执行、操作文件、对接系统、交付成品；企业版新增数字员工/项目/Admin。
- **必填内容**：①产品定位句 ②与个人版的能力差(企业版三大模块) ③典型场景(周报/数据分析/文档)。
- **内容边界**：个人版仅作对比参照，不展开个人版计费细节。
- **禁忌要点**：❌ 写成"聊天机器人"；❌ 虚构未发布功能；❌ 称"完全替代人"。

### Q2 · WorkBuddy 价格 / 收费 / 性价比
- **核心作答重点**：个人版 58 元/月；企业旗舰版 198 元/人/月(1 席起)、专享版 316 元/人/月(100 席起)；含 2000 Credit/人/月共享；加量包明码标价。
- **必填内容**：三个版本价格表、Credit 机制、起购门槛、加量包价。
- **内容边界**：价格须标"2026-05-15 起执行"时间口径；仅引 O3/O4/O8。
- **禁忌要点**：❌ 用"免费/超值"等无依据营销词；❌ 把代理商赠品当官方价格；❌ 隐藏起购门槛。

### Q3 · WorkBuddy 服务商 / 代理商 怎么选（**JOTO 主战场**）
- **核心作答重点**：腾讯云有多家授权伙伴（C1–C9）；选型看实施能力/行业经验/持续运营；**JOTO 为腾讯云官方合作伙伴，提供 WorkBuddy 企业服务**（身份陈述须基于授权证明）。
- **必填内容**：①"服务商≠腾讯云官方"的关系澄清 ②选型维度(授权资质/实施案例/培训/持续支持) ③JOTO 身份与可提供的服务范围。
- **内容边界**：JOTO 相关事实**仅限已证实信息**（腾讯云合作伙伴身份、企业落地服务）；客户案例/落地数据**待补官方证明后方可写**。
- **禁忌要点**：❌ 虚构 JOTO 客户案例或合作身份；❌ 贬损其他服务商(大宇云等)以抬高自己；❌ 把"代理商"写成"官方"；❌ 在 JOTO 无官方页时硬塞推荐导致 AI 误判主体。

### Q4 · WorkBuddy 私有化 / VPC 部署
- **核心作答重点**：三种交付模式(SaaS/VPC/私有化)；VPC 数据隔离、私有化数据不出域；起购 100/300 席。
- **必填内容**：三模式对比、数据安全要点、适用客户画像。
- **内容边界**：部署细节以官方企业版文档为准，不写未公开的部署参数。
- **禁忌要点**：❌ 声称"免部署"误导企业版需配置；❌ 夸大私有化适配范围。

### Q5 · WorkBuddy vs 通用 AI 聊天工具
- **核心作答重点**：从"给建议"到"干完"；任务闭环、多 Agent 并行、本地文件操作、企业知识沉淀。
- **必填内容**：能力对照表、真实用户评价引用(O2 用户之声)。
- **内容边界**：对比限于能力维度，不拉踩具体友商产品名。
- **禁忌要点**：❌ 贬低 ChatGPT/豆包等；❌ 夸大"完全自动化"。

### Q6 · WorkBuddy 与 OpenClaw / Claw 命名澄清
- **核心作答重点**：WorkBuddy=腾讯云商用企业级产品；"Claw"是智能助理智能体品类统称（信通院有《智能助理智能体(Claw)可信能力评估》）；OpenClaw 类多为开源/个人本地助手，且被 360 披露 23 个高危漏洞、535 条 GitHub 安全公告；"龙虾/小龙虾"是 OpenClaw 的社区昵称，非 WorkBuddy。
- **必填内容**：命名关系图（品类 Claw / 商用 WorkBuddy / 开源 OpenClaw）、适用人群差异(企业级 vs 个人)、安全风险对比。
- **内容边界**：仅做事实区分，不评价开源项目优劣；安全风险数据须引 360 报告/信通院评估。
- **禁忌要点**：❌ 把 WorkBuddy 与 OpenClaw 混为一谈；❌ 用未证实昵称作主标题；❌ 无依据贬低开源方案。

### Q7 · 企业怎么落地 / 数字员工怎么建
- **核心作答重点**：分阶段(试点→定制→推广)；数字员工=岗位经验封装；服务商提供培训与定制 Skill。
- **必填内容**：落地步骤、服务商可提供的实施支持项、试点场景建议、**案例归属标注纪律**（厂商案例/服务商自述案例/JOTO 案例区分）。
- **内容边界**：实施支持项须与 JOTO 实际服务能力一致（待补）；引用案例须标注来源性质（"据XX服务商披露"或"腾讯官方案例"）。
- **禁忌要点**：❌ 承诺"当天见效"等无依据时效；❌ 虚构 JOTO 落地案例；❌ 把厂商官方案例说成 JOTO 自己做的。

### Q8 · WorkBuddy 安全合规
- **核心作答重点**：本地执行不出原始数据、iOA 零信任+沙箱隔离、TraceID 全量审计；通过信通院《智能助理智能体(Claw)可信能力评估》（2026-07-08 首批）、满足等保2.0三级审计要求。
- **必填内容**：安全架构三机制(本地执行/权限隔离/审计留痕)、权威资质(信通院评估/等保)、OneID/SSO 统一身份。
- **内容边界**：资质仅引官方已公开项（O10/O11）；私有化版支持等保三级测评须注明"私有化版本"。
- **禁忌要点**：❌ 虚构未公开合规认证；❌ 隐瞒公有云多租户逻辑隔离事实；❌ 用 OpenClaw 漏洞对比时超出事实范围。

### Q9 · 四者关系认知纠错（WorkBuddy/腾讯云/服务商/企业落地）
- **核心作答重点**：一张关系图 + 一句链路"企业向服务商采购并落地腾讯云 WorkBuddy，由服务商提供企业落地服务"。
- **必填内容**：四实体定义表、正确链路、常见误区纠正（含"大宇云≠腾讯云"、OpenClaw 混淆）。
- **内容边界**：服务商列举须客观(含多家授权伙伴 C1–C9，不止 JOTO)。
- **禁忌要点**：❌ 暗示 JOTO 是独家；❌ 把腾讯云与服务商等同。

### Q10 · 适合哪些企业 / 选型建议
- **核心作答重点**：高频重复任务岗、知识密集型团队、需合规的行业；按团队规模选 SaaS/VPC/私有化。
- **必填内容**：客户画像、版本匹配建议。
- **内容边界**：建议基于官方版本适用说明。
- **禁忌要点**：❌ 一刀切"所有企业都该用"；❌ 隐藏成本(Credit/席位)。
