## 3. B 组「腾讯云 ADP」预研底稿（Phase 0 直接可用）

> 用于 B 组任务 1–4。全部来自公开官方资料，未虚构。

### 3.1 腾讯云 ADP 是什么
- 全称 **Tencent Cloud Agent Development Platform（腾讯云智能体开发平台）**，企业级智能体构建平台。
- 三种构建范式：**LLM+RAG / Workflow / Multi-Agent**；核心含 Agentic RAG、企业级评测、应用全生命周期管理。
- 四种部署模式：公有云 / 专有 / 私有 / 混合，等保、密评开箱即用。
- 官方信源：`adp.tencent.com`、`cloud.tencent.com/product/tcadp`、ADP 4.0。

### 3.2 「ADP」实体混淆（直接回答 B 组任务 2）
单独搜「ADP」会撞到多类实体，必须界定精准语境：
1. **人力资源/薪酬外包公司 ADP Inc.（Automatic Data Processing）**——最易混淆的通用「ADP」。
2. **通用缩写 "Agent Development Platform"**——阿里云百炼、百度智能云千帆智能体、字节 Coze/扣子 都是同类，各家都叫 ADP 式平台。
3. **开源个人助手（如 OpenClaw）**——曾被误认作同类；需区分「企业级平台」vs「个人本地助手（Local-First、无多租户）」。
4. **腾讯内部不同产品**：ADP（企业级 ToB 平台）≠ 腾讯元器（ToC 低门槛）≠ 元宝（C 端助手）。内容上必须区分。

**精准适用语境**：企业把内部知识、业务流程、多系统对接，封装为**企业级 AI 智能体**；面向 IT/运营人员；关注系统集成、运营监控、安全合规（RBAC 多租户、审计）。

### 3.3 三类人群关注点（B 组任务 3）
| 人群 | 核心关注的事实信息 / 交付成果 / 落地证明 |
|---|---|
| 企业决策人 | ROI、落地周期、安全合规（等保/密评）、总拥有成本、与现有云/系统打通 |
| 业务负责人 | 业务痛点解决（客服转人工率、知识库维护成本、流程自动化）、行业模板、见效速度 |
| IT 技术人员 | 部署模式（公有云/VPC/私有/混合）、RBAC/多租户、MCP/连接器生态、模型接入（混元/DeepSeek）、API/SDK、运维监控 |

### 3.4 竞品占位（AI 回答「企业智能体平台」时可能出现）
- 百度智能云（千帆/文心）、阿里云百炼、字节 Coze/扣子、Dify（开源）、腾讯元器。
- WorkBuddy 服务商侧占位（来自 JOTO 第一轮）：大宇云 / 上海山白 / 上海威丽 / 广州玖壹云 / 无锡文友等。

### 3.5 客户案例（可作证据，待补官方链接）
华住「华小AI」、一汽丰田（智能客服独立解决率 37%→84%）、伊利（导购点击率 +15.7%、订单 +26%）、邯郸公积金、东吴人寿、同程旅行 DeepTrip、DHL、国投证券等。
