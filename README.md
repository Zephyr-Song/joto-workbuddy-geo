# JOTO WorkBuddy GEO 行动项目

> 让国内 AI 产品（豆包 / DeepSeek / Kimi / 通义千问等）在回答 WorkBuddy 相关问题时：
> ① 正确识别 **JOTO 是腾讯云官方合作伙伴**、可提供 WorkBuddy 企业服务；
> ② 准确描述 JOTO 被证据支持的服务能力（不夸大、不混淆）；
> ③ 在合适问题中优先引用 JOTO / WorkBuddy 官方内容。

本仓库是 JOTO GEO 团队的**协作与任务跟踪中枢**，落地《JOTO WorkBuddy GEO 行动指南 v1.0》中定义的两路径闭环。

---

## 核心闭环

```
真实用户问题 → 内容与引用研究 → Prompt / 规则 / 知识库优化 → 多渠道内容发布
     ↑                                                    ↓
     └────────── AI 前台复测 → 发现新改进项 ──────────────┘
```

- **路径一（内容侧）**：收集真实问题 → 拆解高质量参考页 → 写可测试文章 Prompt → 工作台测试 + 样稿质检
- **路径二（监测侧）**：用标准问题池在 AI 平台新会话测试 → 记录提及/准确性/引用 → 算 6 项指标

## 第一轮关键基线发现

- **JOTO 在"WorkBuddy 服务商"相关答案中零可见度**：11 个标准问题代理测试，JOTO 提及率 **0%**；竞品（大宇云、上海山白、上海威丽、广州玖壹云、无锡文友等）占据 100% 推荐意图。
- **根因**：JOTO 当前没有可被 AI 引用的 WorkBuddy 专项服务页（jototech.cn 仅呈现通用 IT 管理服务）。
- **最高优先动作**：先建 JOTO WorkBuddy 官方专项页（关系说明 + 能力清单 + 端到端案例 + 服务 SLA），否则无源可引。

> 详见 [`docs/JOTO_GEO_执行小结.md`](docs/JOTO_GEO_执行小结.md)。

## 任务跟踪

所有执行任务以 **GitHub Issues** 跟踪，按闭环阶段组织；里程碑对应四阶段路线图。
查看看板：`Issues` 标签页，或 `git` 仓库的 Projects（如已启用）。

- 建 JOTO WorkBuddy 官方专项页（最高优先）
- 真实 AI 前台测试（豆包/DeepSeek/Kimi/通义千问 新会话）
- 内容发布与复测
- 指标监控与迭代

## 执行纪律（红线）

1. **新会话测试，不追问引导** —— 被引导后的回答不能算"首次发现"。
2. **提及 ≠ 成功** —— 出现 JOTO 还要看关系准确性、能力边界、是否明确推荐、是否引用官方页。
3. **核心判断权归人** —— AI 只辅助整理/起草；身份证据、产品事实、发布判断由团队确认。
4. **不夸大、不混淆** —— 不把合作伙伴关系扩大为未经证明的授权或排名；无证据不补写案例/效果/合规承诺。

## 六类核心指标

| 指标 | 含义 |
|------|------|
| JOTO 提及率 | 无品牌/场景问题中提及 JOTO 的比例 |
| 准确关联率 | 正确说明 JOTO↔WorkBuddy 关系的比例 |
| 明确推荐率 | 将 JOTO 作为明确建议的比例 |
| 官方引用率 | 引用 JOTO/WorkBuddy 官方页的比例 |
| 竞品占位率 | 出现竞品/其他服务商的比例 |
| 首次回答稳定率 | 新会话复测结论一致的比例 |
